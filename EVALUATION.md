# Canonical Evaluation Flow

This repository now emits stable benchmark artifacts for both inference stages so performance claims can be audited after the fact.

## Required Metrics

For headline quality claims, report:

- `dice`
- `bdice_10mm`
- `hd95_mm`
- `runtime_sec`
- `gpu_peak_memory_mb`

Use the aggregated summaries generated by the scripts, not ad hoc notebook calculations.

## Required Artifacts

### Stage 1: Point-Cloud Inference

Run `pcdiff/test_completion.py` as usual. In addition to the per-case `syn/` outputs, the run now writes:

- `benchmark_cases.csv`
- `benchmark_summary.json`
- `benchmark_stage_report.json`

These artifacts live under `--eval_path`.

### Stage 2: Voxelization + Metric Computation

Run `voxelization/generate.py` as usual. In addition to per-case `mean_impl.nrrd` and `eval_metrics.yaml`, the run now writes:

- `benchmark_cases.csv`
- `benchmark_summary.json`
- `benchmark_stage_report.json`

These artifacts live under `train.out_dir/generation.generation_dir`.

Stage-2 retraining manifests may now target either legacy `*_pc.npz` inputs or generated stage-1 completions:

- legacy row: `path/to/case_base`
- explicit row: `path/to/input_pc.npz,path/to/target_vox.npz`
- generated completion row: `path/to/sample.npy,path/to/shift.npy,path/to/scale.npy,path/to/target_vox.npz[,ensemble_index]`

The default stage-2 inference postprocess is now defect-aware and config-driven via `generation.postprocess.*` instead of the previous hard-coded spherical crop.

An experimental stage-2 replacement path is also available via `generation.refinement.strategy=occupancy_symmetry`. It switches the final complete-skull reconstruction from legacy majority vote to occupancy-probability thresholding with an optional defect-local symmetry prior before implant extraction. Keep benchmark reports for this path separate from the legacy stage-2 summaries.

## Claim Discipline

Any benchmark claim should retain:

- the stage 1 report
- the stage 2 report
- the model checkpoint paths used by both stages
- the exact config or CLI arguments
- the git commit recorded in each stage report

If any of those are missing, treat the result as exploratory rather than canonical.

## Recommended Procedure

1. Run point-cloud inference with a dedicated `--eval_path`.
2. Run voxelization using that output as `data.path`.
3. Archive both stage reports together with the referenced checkpoints.
4. Quote metrics from `benchmark_summary.json`, and use `benchmark_cases.csv` for per-case analysis.
